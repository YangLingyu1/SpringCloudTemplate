# 2026-04-10

## 修复 PaginationInnerInterceptor 找不到的问题
- 问题：MP 3.5.15 + Spring Boot 4.0.3，`PaginationInnerInterceptor` 无法解析
- 原因：MP v3.5.9 起分页插件从核心模块分离，需额外引入 `mybatis-plus-jsqlparser`
- 修复：在 `common/pom.xml` 添加 `mybatis-plus-jsqlparser:${mybatis-plus.version}` 依赖

## user-service 启动失败：DataSource 未配置
- 问题：user-service 启动报 `Failed to configure a DataSource: 'url' attribute is not specified`
- 根本原因：order-service 的 bootstrap.yml import 拉取的是 `user-service` 配置而不是 `order-service`，导致 order-service 蹭到 user-service 的数据库配置能启动；而 Nacos 上 user-service 配置缺少 datasource，user-service 本身启动失败
- 修复：将 order-service/bootstrap.yml 中 `nacos:user-service` 改为 `nacos:order-service`；并需在 Nacos 控制台的 user-service 配置中补充 spring.datasource 相关配置

## Nacos 配置 DataID 命名规范
- bootstrap.yml 的 import 中 `nacos:xxx` 与 Nacos 控制台的 Data ID 必须完全一致（含 .yml 后缀）
- 推荐统一使用 `.yml` 后缀：Data ID 命名为 `common.yml`、`user-service.yml`、`order-service.yml`
- bootstrap.yml 中需同时设置 `file-extension: yml` 和 import 里加 `.yml`
- datasource 等服务私有配置放在各服务自己的 Data ID，公共配置放 common.yml
